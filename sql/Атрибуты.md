# **`PRIMARY KEY` (Первичный ключ)**

Это атрибут, который уникально идентифицирует каждую строку в таблице. Каждый столбец или несколько столбцов могут быть первичным ключом, и его значения должны быть уникальны.

**Пример:**

```sql
USE productsdb;

CREATE TABLE Customers (
    Id INT PRIMARY KEY,
    Age INT,
    FirstName VARCHAR(20),
    LastName VARCHAR(20)
);
```

## **Составной первичный ключ**

Первичный ключ может включать несколько столбцов. Такой ключ называется составным и гарантирует, что комбинация значений указанных столбцов будет уникальной.

**Пример:**

```sql
CREATE TABLE OrderLines (
    OrderId INT,
    ProductId INT,
    Quantity INT,
    Price MONEY,
    PRIMARY KEY(OrderId, ProductId)
);
```

# `**AUTO_INCREMENT**`
Атрибут `AUTO_INCREMENT` автоматически увеличивает значение столбца при добавлении новых строк. Используется для столбцов с целочисленным типом данных.

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT PRIMARY KEY AUTO_INCREMENT,
    Age INT,
    FirstName VARCHAR(20),
    LastName VARCHAR(20)
);
```

# `**UNIQUE**`

Атрибут `UNIQUE` указывает, что значения столбца должны быть уникальными. В одной таблице не может быть двух строк с одинаковым значением в таком столбце.

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT PRIMARY KEY AUTO_INCREMENT,
    Age INT,
    FirstName VARCHAR(20),
    LastName VARCHAR(20),
    Phone VARCHAR(13) UNIQUE
);
```

В этом случае столбец `Phone` не может содержать повторяющихся значений.

Также можно определить уникальность на уровне таблицы.

```sql
CREATE TABLE Customers (
    Id INT PRIMARY KEY AUTO_INCREMENT,
    Age INT,
    FirstName VARCHAR(20),
    LastName VARCHAR(20),
    Email VARCHAR(30),
    Phone VARCHAR(20),
    UNIQUE(Email, Phone)
);
```

# **`NULL` и `NOT NULL`**

Атрибут `NOT NULL` указывает, что столбец не может содержать пустые значения (`NULL`). Если атрибут не указан, по умолчанию столбец допускает `NULL`.

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT PRIMARY KEY AUTO_INCREMENT,
    Age INT,
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    Email VARCHAR(30) NULL,
    Phone VARCHAR(20) NULL
);
```

# `**DEFAULT**`

Атрибут `DEFAULT` задает значение по умолчанию для столбца, если при добавлении строки в него не указано значение.

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT PRIMARY KEY AUTO_INCREMENT,
    Age INT DEFAULT 18,
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    Email VARCHAR(30) NOT NULL UNIQUE,
    Phone VARCHAR(20) NOT NULL UNIQUE
);
```

# **`CHECK`**

Атрибут `CHECK` задает ограничение на значения, которые могут храниться в столбце.

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT AUTO_INCREMENT,
    Age INT DEFAULT 18 CHECK(Age > 0 AND Age < 100),
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    Email VARCHAR(30) CHECK(Email != ''),
    Phone VARCHAR(20) CHECK(Phone != '')
);
```

Проверка `CHECK` гарантирует, что `Age` будет только в диапазоне от 0 до 100, а `Email` и `Phone` не будут пустыми строками.

Также можно задавать ограничения на уровне таблицы:

```sql
CREATE TABLE Customers (
    Id INT AUTO_INCREMENT,
    Age INT DEFAULT 18,
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    Email VARCHAR(30),
    Phone VARCHAR(20),
    CHECK ((Age > 0 AND Age < 100) AND (Email != '') AND (Phone != ''))
);
```

# **`CONSTRAINT` (Именование ограничений)**

Ключевое слово `CONSTRAINT` позволяет задавать имена для ограничений. Это может пригодиться для управления ограничениями в будущем (удаление или изменение).

**Пример:**

```sql
CREATE TABLE Customers (
    Id INT AUTO_INCREMENT,
    Age INT,
    FirstName VARCHAR(20) NOT NULL,
    LastName VARCHAR(20) NOT NULL,
    Email VARCHAR(30),
    Phone VARCHAR(20) NOT NULL,
    CONSTRAINT customers_pk PRIMARY KEY(Id),
    CONSTRAINT customer_phone_uq UNIQUE(Phone),
    CONSTRAINT customer_age_chk CHECK(Age > 0 AND Age < 100)
);
```

Здесь заданы имена для каждого ограничения:

- `customers_pk` для `PRIMARY KEY`,
- `customer_phone_uq` для `UNIQUE`,
- `customer_age_chk` для `CHECK`.

# Внешние ключи `FOREIGN KEY`

Внешние ключи в SQL используются для создания связей между таблицами. Внешний ключ определяет зависимость между столбцами одной таблицы и столбцами другой таблицы. Обычно внешний ключ ссылается на `PRIMARY KEY` в связанной таблице.

```sql
FOREIGN KEY (столбец_зависимой_таблицы)
REFERENCES главная_таблица (столбец_главной_таблицы)
```

Здесь `столбец_зависимой_таблицы` указывает на ключ в другой таблице, а `REFERENCES` связывает его с главным столбцом в другой таблице.

**Пример:**

```sql

CREATE TABLE Departments (
    DepartmentID INT PRIMARY KEY,
    DepartmentName VARCHAR(50)
);

CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    Name VARCHAR(50),
    DepartmentID INT,
    FOREIGN KEY (DepartmentID) REFERENCES Departments (DepartmentID)
);
```

Здесь таблица `Employees` ссылается на таблицу `Departments`, используя внешний ключ `DepartmentID`. То есть каждый сотрудник относится к определенному отделу.

# `ON DELETE` и `ON UPDATE`

С помощью параметров `ON DELETE` и `ON UPDATE` можно настроить поведение при удалении или изменении строки в главной таблице.

- `CASCADE` - автоматически удаляет или изменяет связанные строки в зависимой таблице при изменении или удалении записи в главной таблице.
- `SET NULL` присваивает `NULL` столбцу внешнего ключа при удалении или изменении связанной строки.
- `RESTRICT` и `NO ACTION` - предотвращают удаление или изменение строки, если в зависимой таблице есть связанные строки.

## Пример с `ON DELETE CASCADE`:

```sql

CREATE TABLE Tasks (
    TaskID INT PRIMARY KEY,
    EmployeeID INT,
    TaskDescription VARCHAR(100),
    FOREIGN KEY (EmployeeID) 
    REFERENCES Employees (EmployeeID)
    ON DELETE CASCADE
);
```

При удалении сотрудника из таблицы `Employees` будут автоматически удаляться все задачи, связанные с этим сотрудником в таблице `Tasks`.

## Пример с `ON DELETE SET NULL`:

Если необходимо сохранить запись, но удалить зависимость, можно использовать `SET NULL`, но при этом столбец должен поддерживать `NULL` значения:

```sql

CREATE TABLE Projects (
    ProjectID INT PRIMARY KEY,
    EmployeeID INT,
    ProjectName VARCHAR(100),
    FOREIGN KEY (EmployeeID) 
    REFERENCES Employees (EmployeeID)
    ON DELETE SET NULL
);
```

В этом случае при удалении сотрудника из таблицы `Employees`, значение столбца `EmployeeID` в таблице `Projects` станет `NULL`, но сама запись проекта останется.
