В SQL система управления базами данных (СУБД) использует права доступа для управления тем, кто и какие операции может выполнять с базой данных. Команда `GRANT` используется для предоставления этих прав пользователям или ролям.

```sql
GRANT [права] ON [объект] TO [пользователь];
```

- права — список прав доступа, которые предоставляются. Например, `SELECT`, `INSERT`, `UPDATE`.
- объект — объект базы данных, на который даются права (например, таблица, представление).
- пользователь — имя пользователя или роли, которой даются права.

**Типы прав доступа:**

- `SELECT`: право на чтение данных из таблицы.
- `INSERT`: право на добавление новых строк в таблицу.
- `UPDATE`: право на изменение данных в таблице.
- `DELETE`: право на удаление строк из таблицы.
- `CREATE`: право на создание новых объектов базы данных.
- `ALTER`: право на изменение структуры существующих объектов (например, добавление столбцов в таблицу).
- `DROP`: право на удаление объектов базы данных (например, таблиц, представлений).
- `REFERENCES`: право на создание внешних ключей.
- `ALL`: предоставляет все права на объект.

**Пример:**

```sql
GRANT SELECT, UPDATE ON employees TO john_doe, jane_smith;
FLUSH PRIVILEGES;
```

В конце используйте `FLUSH PRIVILEGES` для обновления и применения изменений, внесенных в таблицы прав доступа.
