# Добавление данных

Добавление данных в таблицу базы данных MySQL осуществляется с помощью команды `INSERT INTO`. Эта команда позволяет вставлять данные как в отдельные строки, так и в несколько строк одновременно.

```sql
INSERT INTO table_name (column1, column2, column3, ...)
VALUES
    (value1_1, value1_2, value1_3, ...),
    (value2_1, value2_2, value2_3, ...),
    ...;
```

**Пример:**

```sql
CREATE TABLE Employees (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    Age INT
);

INSERT INTO Employees (FirstName, LastName, Age)
VALUES
    ('Alice', 'Smith', 28),
    ('Bob', 'Johnson', 35),
    ('Charlie', 'Lee', 40);
```

## Добавление данных в определённые столбцы

Если значения нужно добавить только в отдельные столбцы, можно указать их список. Для остальных столбцов будут установлены значения по умолчанию или `NULL` (если это разрешено).

**Пример:**

```sql
INSERT INTO Employees (FirstName, LastName)
VALUES ('Emily', 'Davis');
```

Здесь будет добавлен новый сотрудник с именем Эмили Дэвис, а столбец `Age` будет содержать `NULL`, если для него разрешены такие значения.

## Добавление данных с использованием `SET`

Вместо перечисления значений через `VALUES`, можно использовать `SET`, чтобы явно присвоить значения определённым столбцам.

**Пример:**

```sql
INSERT INTO Employees
SET FirstName = 'Frank', LastName = 'Taylor', Age = 27;
```

## Использование `DEFAULT` для значений по умолчанию

Если нужно явно указать, что значение столбца должно быть значением по умолчанию, можно использовать ключевое слово `DEFAULT`.

**Пример:**

```sql
INSERT INTO Employees (FirstName, LastName, Age)
VALUES ('George', 'Brown', DEFAULT);
```

## `INSERT IGNORE`

Если нужно избежать ошибок при вставке данных, таких как дублирование уникальных значений, можно использовать `INSERT IGNORE`. В этом случае MySQL проигнорирует строки, которые вызывают ошибки.

**Пример:**

```sql
INSERT IGNORE INTO Employees (Id, FirstName, LastName, Age)
VALUES (1, 'Henry', 'Ford', 45);
```

Если запись с `Id = 1` уже существует, она будет пропущена, и ошибка не возникнет.

## `ON DUPLICATE KEY UPDATE`

Эта опция позволяет обновлять существующую запись, если нарушено уникальное или первичное ключевое ограничение. Вместо вставки дублирующей записи, уже существующая строка будет обновлена.

```sql
INSERT INTO table_name (column1, column2, ...)
VALUES (value1, value2, ...)
ON DUPLICATE KEY UPDATE column1 = new_value1, column2 = new_value2;
```

##  Добавление данных из другого запроса🌟

Вы можете добавить данные в таблицу не только вручную, но и скопировать их из другой таблицы или результата запроса.

```sql
INSERT INTO table_name (column1, column2, column3, ...)
SELECT expression1, expression2, expression3, ...
FROM another_table
WHERE conditions;
```

**Пример:**

```sql
--Предположим, у нас есть таблица ArchivedEmployees, 
--и нам нужно скопировать из неё данные в таблицу Employees:

INSERT INTO Employees (FirstName, LastName, Age)
SELECT FirstName, LastName, Age
FROM ArchivedEmployees
WHERE Age > 30;
```

## Вставка данных с помощью подзапросов

Можно использовать подзапросы для динамической вставки данных на основе других условий.

**Пример:**

```sql
INSERT INTO Employees (FirstName, LastName, Age)
VALUES ((SELECT FirstName FROM Employees WHERE Id = 1), 'Jones', 40);

--Здесь мы используем подзапрос для вставки имени из другой записи таблицы Employees.
```

# Удаление данных

## Удаление данных из таблиц

Для удаления данных из БД используется команда `DELETE`. Она позволяет удалять строки, соответствующие заданным условиям.

```sql
DELETE FROM table_name
WHERE condition;
```

**Пример:**

```sql
DELETE FROM Employees
WHERE Age > 40;
```

В данном примере будут удалены все сотрудники, чей возраст больше 40 лет.

## Удаление всех данных из таблицы

Если не указано условие `WHERE`, команда `DELETE` удалит все строки в таблице.

**Пример:**

```sql
DELETE FROM Employees;
```

Важно быть осторожным, так как после удаления всех строк восстановить их будет невозможно. Для очистки таблицы быстрее использовать команду `TRUNCATE`:

```sql
TRUNCATE TABLE Employees;
```

## Удаление с ограничением по количеству строк

Чтобы ограничить количество удаляемых строк, можно использовать ключевое слово `LIMIT`.

**Пример:**

```sql
DELETE FROM Employees
WHERE Age > 40
LIMIT 5;
```


## Удаление с каскадным удалением

Когда таблицы связаны внешними ключами с каскадным удалением (`ON DELETE CASCADE`), удаление записей в одной таблице автоматически удаляет связанные записи в другой.

**Пример:**

```sql
CREATE TABLE customers (
    customer_id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100)
);


CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    customer_id INT,
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id) ON DELETE CASCADE
);


INSERT INTO customers (name) VALUES ('Иван'), ('Мария');
INSERT INTO orders (customer_id, order_date) VALUES (1, '2024-01-01'), (1, '2024-02-01'), (2, '2024-03-01');


--Удаляем клиента с customer_id = 1
DELETE FROM customers WHERE customer_id = 1;

--Клиент с customer_id = 1 удалён из таблицы customers.
--Все заказы, связанные с этим клиентом, автоматически удалены из таблицы orders.
```

## Удаление с различными командами: `DELETE`, `TRUNCATE`, `DROP`

`DELETE`, `TRUNCATE` и `DROP` — три команды для удаления данных и таблиц, каждая из которых имеет свои особенности.

**Пример:**

```sql
DELETE FROM Employees WHERE Age > 40;
TRUNCATE TABLE Employees;
DROP TABLE Employees;
```

`DELETE` удаляет строки, `TRUNCATE` очищает таблицу, а `DROP` полностью удаляет таблицу.

## Удаление с использованием соединений (`JOIN`)🌟

Удалять строки можно на основе условий в другой таблице с использованием `JOIN`.

**Пример:**

```sql
DELETE Orders
FROM Orders
JOIN Customers ON Orders.CustomerId = Customers.Id
WHERE Customers.Age > 50;
```

Этот запрос удаляет заказы клиентов старше 50 лет.

## Удаление с использованием подзапросов

Вы также можете использовать подзапросы для удаления строк, основываясь на результатах других запросов.

**Пример:**

```sql
DELETE FROM Orders
WHERE CustomerId IN (
    SELECT Id FROM DeletedCustomers
);
```

# Обновление данных

Для обновления данных в БД используется команда `UPDATE`. Основной синтаксис команды следующий:

```sql
UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;
```

**Пример:**

```sql
UPDATE Employees
SET Age = 30
WHERE FirstName = 'Alice' AND LastName = 'Smith';
```

## Обновление нескольких столбцов

Вы можете обновлять сразу несколько столбцов в одном запросе. Для этого их нужно указать через запятую.

**Пример:**

```sql
UPDATE Employees
SET FirstName = 'Bob', Age = 35
WHERE Id = 2;
```

Этот запрос обновляет имя и возраст сотрудника с `Id = 2`.

## Обновление всех записей таблицы

Если вы не укажете условие `WHERE`, команда `UPDATE` применит изменения ко всем записям таблицы.

**Пример:**

```sql
UPDATE Employees
SET Age = 25;
```


## Обновление с использованием `JOIN`🌟

Можно выполнять обновления с использованием `JOIN` для работы с несколькими таблицами одновременно.

**Пример:**

```sql
UPDATE Employees
JOIN Departments ON Employees.DepartmentId = Departments.Id
SET Employees.Salary = 50000
WHERE Departments.Name = 'HR';
```

Этот запрос обновляет зарплату сотрудников в отделе «HR» до 50,000.

##  Обновление данных с использованием подзапросов

В запросе `UPDATE` можно использовать подзапросы для обновления данных на основе результатов других запросов. Это позволяет более гибко управлять обновлениями, например, когда данные в одной таблице зависят от значений в другой.

**Пример:**

```sql
UPDATE Employees
SET Age = (SELECT MAX(Age) FROM Employees WHERE Department = 'IT')
WHERE Department = 'Finance';
```
