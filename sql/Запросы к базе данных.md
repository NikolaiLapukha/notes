# Команда `SELECT`

Для выборки данных используется команда `SELECT`. Ее базовый синтаксис выглядит следующим образом:

```sql
SELECT список_столбцов FROM название_таблицы;
```

**Пример:**

```sql
SELECT * FROM Books; -- выбрать все столбцы из таблицы Books
```

 **Выбор отдельных столбцов**

Для того чтобы выбрать конкретные столбцы, их нужно указать через запятую.

**Пример:**

```sql
SELECT Title, Price FROM Books;

/*
+-------------------------------+--------+
| Title                         | Price  |
+-------------------------------+--------+
| The Catcher in the Rye        | 450.00 |
| 1984                          | 350.00 |
| Brave New World               | 400.00 |
| Fahrenheit 451                | 300.00 |
| To Kill a Mockingbird         | 500.00 |
...
*/
```

# Псевдонимы для столбцов

С помощью оператора `AS` можно задавать имена для столбцов в результате выборки.

**Пример:**

```sql
SELECT Title AS BookTitle, Price AS BookPrice FROM Books;

/*
+-------------------------------+-----------+
| BookTitle                     | BookPrice |
+-------------------------------+-----------+
| The Catcher in the Rye        |    450.00 |
| 1984                          |    350.00 |
| Brave New World               |    400.00 |
| Fahrenheit 451                |    300.00 |
| To Kill a Mockingbird         |    500.00 |
...
*/
```

# Псевдонимы без использования `AS` и обращение через точку

В SQL не всегда требуется использовать оператор `AS` для задания псевдонима таблицы. Вместо этого псевдонимы для таблиц можно задавать непосредственно после имени таблицы в запросе, и это часто используется для упрощения длинных запросов или при работе с несколькими таблицами. После присвоения псевдонима, можно обращаться к столбцам таблицы через псевдоним с помощью точки (`.`).

**Пример:**

```sql
--Запрос вернет заголовки книг и цены, если цена больше 600.

FROM Books B
WHERE B.Price > 600;
```

# Оператор `DISTINCT`

Оператор `DISTINCT` в SQL используется для выборки уникальных значений из столбцов, исключая повторяющиеся записи. Это особенно полезно, когда нужно получить список всех различных значений в одном или нескольких столбцах, игнорируя дубликаты.

```sql
SELECT DISTINCT столбец_1, столбец_2, ... FROM таблица;
```

# Оператор фильтрации `IN`

Оператор `IN` в SQL используется для фильтрации данных по списку значений. Он позволяет выбрать записи, значения которых совпадают с одним из указанных в списке значений. Это упрощает написание запросов, где нужно указать несколько условий для одного столбца.

```sql
SELECT столбцы 
FROM таблица 
WHERE столбец IN (значение1, значение2, значение3, ...);
```

**Пример:**

```sql
--Мы хотим выбрать все книги, автором которых является или Джордж Оруэлл (George Orwell), или Рэй Брэдбери (Ray Bradbury).
SELECT * FROM Books
WHERE Author IN ('George Orwell', 'Ray Bradbury');

/*
+----+----------------+---------------+--------+---------+
| Id | Title          | Author        | Price  | InStock |
+----+----------------+---------------+--------+---------+
|  2 | 1984           | George Orwell | 350.00 |       3 |
|  4 | Fahrenheit 451 | Ray Bradbury  | 300.00 |       0 |
|  6 | Animal Farm    | George Orwell | 250.00 |       4 |
+----+----------------+---------------+--------+---------+
*/
```

# Использование оператора `NOT IN`

Оператор `NOT IN` является противоположностью оператора `IN` и возвращает строки, значения которых **не находятся** в указанном списке.

**Пример:**

```sql
--мы хотим выбрать все книги, автором которых не является Джордж Оруэлл.
SELECT * FROM Books
WHERE Author NOT IN ('George Orwell');

/*
+----+-------------------------------+------------------------+--------+---------+
```


# Оператор `BETWEEN`

Оператор `BETWEEN` используется для выборки данных, которые находятся в пределах определённого диапазона. Этот диапазон может включать числовые значения, даты или даже текстовые строки. Оператор проверяет, попадает ли значение в указанный диапазон, включая его границы.

```sql
SELECT столбцы
FROM таблица
WHERE столбец BETWEEN значение_1 AND значение_2;
```

Оператор `BETWEEN` эквивалентен следующему условию:

```sql
WHERE столбец >= значение_1 AND столбец <= значение_2;
```

**Пример:**

```sql
SELECT Title, Price
FROM Books
```

# Оператор `LIKE`

Оператор `LIKE` используется для поиска строк, которые соответствуют заданному шаблону.

```sql
SELECT столбцы
FROM таблица
WHERE столбец LIKE 'шаблон';
```

Шаблоны могут содержать специальные символы:

- `%` — соответствует любой последовательности символов (включая пустую).
- `_` — соответствует любому одиночному символу.

 **Пример:**

```sql
--Поиск книг, название которых начинается с "The"
SELECT Title
FROM Books
WHERE Title LIKE 'The%';

/*
+----------------------------+
| Title                      |
+----------------------------+
| The Catcher in the Rye     |
| The Great Gatsby           |
| The Odyssey                |
| The Iliad                  |
| The Brothers Karamazov     |
| The Hobbit                 |
| The Lord of the Rings      |
| The Picture of Dorian Gray |
| The Divine Comedy          |
| The Count of Monte Cristo  |
+----------------------------+
*/
```

# Оператор `REGEXP`

Оператор `REGEXP` используется для более сложных регулярных выражений. Он даёт возможность выполнять гибкие поиски, например, поиск по начальной или конечной части строки, группам символов и шаблонам.

```sql
SELECT столбцы
FROM таблица
WHERE столбец REGEXP 'шаблон';
```

**Пример:**

```sql
--Поиск книг, названия которых начинаются с буквы "H"
SELECT Title
FROM Books
WHERE Title REGEXP '^H';

/*
+--------+
| Title  |
+--------+
| Hamlet |
+--------+
*/
```

# Оператор `ORDER BY`

Оператор `ORDER BY` используется для сортировки результатов запроса по одному или нескольким столбцам. Сортировка может быть выполнена по возрастанию (по умолчанию) или по убыванию.

```sql
SELECT столбцы
FROM таблица
ORDER BY столбец [ASC|DESC];
```

- **ASC** — сортировка по возрастанию (от меньшего к большему, используется по умолчанию).
- **DESC** — сортировка по убыванию (от большего к меньшему

# Агрегатные функции

Основные агрегатные функции:

| Функция   | Описание                   |
| --------- | -------------------------- |
| `COUNT()` | Подсчёт количества записей |
| `SUM()`   | Суммирование значений      |
| `AVG()`   | Среднее значение           |
| `MIN()`   | Минимальное значение       |
| `MAX()`   | Максимальное значение      |
Пример:

```sql
--Подсчёт количества записей
SELECT COUNT(*) AS TotalBooks
FROM Books;
/*
+------------+
| TotalBooks |
+------------+
|         30 |
+------------+
*/

--Суммирование значений
SELECT SUM(Price) AS TotalPrice
FROM Books;
/*
+------------+
| TotalPrice |
+------------+
|   14690.00 |
+------------+
*/

--Среднее значение
SELECT AVG(Price) AS AveragePrice
FROM Books;
/*
+--------------+
| AveragePrice |
+--------------+
|   489.666667 |
+--------------+
*/

--Минимальное значение
SELECT MIN(Price) AS LowestPrice
FROM Books;
/*
+-------------+
| LowestPrice |
+-------------+
|      200.00 |
+-------------+
*/

--Максимальное значение
SELECT MAX(Price) AS HighestPrice
FROM Books;
/*
+--------------+
| HighestPrice |
+--------------+
|       800.00 |
+--------------+
*/
```

# Условная агрегатная функция с `HAVING`

Оператор `HAVING` позволяет фильтровать группы, которые были созданы с помощью `GROUP BY`. Он работает аналогично оператору `WHERE`, но применяется уже к агрегированным данным.

**Пример:**

```sql
SELECT Author, COUNT(*) AS BooksCount
FROM Books
GROUP BY Author
HAVING COUNT(*) > 1;

/*
+-------------------+------------+
| Author            | BooksCount |
+-------------------+------------+
| George Orwell     |          2 |
| Leo Tolstoy       |          2 |
| Fyodor Dostoevsky |          2 |
| Homer             |          2 |
| J.R.R. Tolkien    |          2 |
+-------------------+------------+
*/
```

# Подзапросы

Основные типы подзапросов:

- Скалярные подзапросы — возвращают одно значение.
- Подзапросы, возвращающие одно или несколько значений — используются для сравнения с оператором `IN`, а также с операторами `ANY`, `ALL`, `SOME`.
- Коррелированные подзапросы — зависят от данных основного запроса и выполняются для каждой строки.
- Подзапрос в блоке `FROM` — позволяет создавать временные таблицы для использования в основном запросе.

Подзапрос обычно заключается в круглые скобки и может быть использован в различных местах основного запроса. Вот общий синтаксис:

```sql
SELECT столбцы
FROM таблица
WHERE столбец оператор (подзапрос);
```

## **Оператор `ALL`**

Оператор `ALL` используется для сравнения значения с каждым элементом, возвращаемым подзапросом. Условие будет истинным **только** если оно выполняется для **всех** элементов из подзапроса.

**Пример:**

```sql
--Найдем книги, цена которых больше, чем цена всех книг автора George Orwell:
```

## **Оператор `ANY`**

Оператор `ANY` проверяет, выполняется ли условие хотя бы для **одного** значения из множества, возвращенного подзапросом. Если хотя бы одно из значений удовлетворяет условию, результат будет TRUE.

**Пример:**

```sql
--Найдем книги, чья цена выше хотя бы одной книги автора Leo Tolstoy:
SELECT Title, Price
FROM Books
WHERE Price > ANY (SELECT Price FROM Books WHERE Author = 'Leo Tolstoy');

/*
+-----------------------+--------+
| Title                 | Price  |
+-----------------------+--------+
| War and Peace         | 750.00 |
| Les Mis?rables        | 720.00 |
| The Lord of the Rings | 800.00 |
| The Divine Comedy     | 800.00 |
+-----------------------+--------+
*/
```

## **Оператор `SOME`**

Оператор `SOME` является синонимом оператора `ANY` и работает точно так же, выполняя условие, если оно истинно хотя бы для одного значения из подзапроса.

## Коррелированные подзапросы

Коррелированные подзапросы зависят от данных основного запроса. Они выполняются для каждой строки основного запроса и могут быть использованы для сложных условий.

**Пример:**

```sql
--Найдем книги, чья цена выше средней цены книг их автора:
SELECT Title, Price, Author
FROM Books B1
WHERE Price > (SELECT AVG(Price) FROM Books B2 WHERE B1.Author = B2.Author);

/*
+------------------------+--------+-------------------+
| Title                  | Price  | Author            |
+------------------------+--------+-------------------+
| 1984                   | 350.00 | George Orwell     |
| War and Peace          | 750.00 | Leo Tolstoy       |

+------------------------+--------+-------------------+

```

## Подзапрос в блоке `FROM`🌟

Подзапросы могут использоваться в блоке `FROM`, чтобы создать временную таблицу, с которой потом можно работать в основном запросе.

**Пример:**

```sql
--Найдем книги, чья цена выше средней для их автора:
SELECT B.Title, B.Price, AvgPrices.AvgPrice
FROM Books B
JOIN (SELECT Author, AVG(Price) AS AvgPrice FROM Books GROUP BY Author) AvgPrices
ON B.Author = AvgPrices.Author
WHERE B.Price > AvgPrices.AvgPrice;

/*
```

# Оператор `EXISTS`

Оператор `EXISTS` используется для проверки наличия строк, которые возвращает подзапрос. Он возвращает значение `TRUE`, если подзапрос возвращает хотя бы одну строку, и `FALSE`, если подзапрос не возвращает ни одной строки. `EXISTS` часто используется в комбинации с подзапросами в выражении `WHERE` для фильтрации данных.

Основная цель оператора `EXISTS` — не возвращать данные, а проверять наличие записей, подходящих под условия, заданные в подзапросе. Как только найдется хотя бы одна строка, подзапрос завершится, и оператор `EXISTS` вернет `TRUE`.

```sql
SELECT столбцы
FROM таблица
WHERE EXISTS (подзапрос);
```
